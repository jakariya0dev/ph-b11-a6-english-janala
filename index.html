Frequently Asked Questions

Difference Between var, let, and const
var is function-scoped, hoisted, and allows re-declaration, making it prone to bugs; let is block-scoped, hoisted but not initialized, and prevents re-declaration; const is also block-scoped but cannot be reassigned after initialization, ensuring immutability for primitive values.
Difference Between map(), forEach(), and filter()
map() returns a new array after transforming each element, forEach() iterates over elements but does not return a new array, and filter() returns a new array containing only elements that meet a condition.
Arrow Functions vs Regular Functions
Arrow functions have a shorter syntax and do not have their own this, instead inheriting this from the surrounding scope, whereas regular functions dynamically bind this based on how they are called; additionally, arrow functions lack an arguments object.
How JavaScript Promises Work
A JavaScript Promise represents an asynchronous operation with three states—pending, fulfilled, or rejected; it helps manage asynchronous code using .then() for success, .catch() for errors, and can be simplified with async/await for better readability.
How Closures Work in JavaScript
A closure occurs when an inner function retains access to variables from its outer function even after the outer function has executed, enabling private variables and encapsulation in JavaScript.
